## 🚀 快速开始

### 安装

```bash

git clone https://github.com/your-org/claudecode-ui-app.git
cd claudecode-ui-app

# 安装依赖
pnpm install
```

## 💡 核心功能

### 1. AI 对话

**基本对话**：
1. 在底部输入框输入问题
2. 按 `Enter` 发送（`Shift+Enter` 换行）
3. AI 会流式返回响应

**示例对话**：
```
[用户] 如何使用 React Hooks？
[AI] React Hooks 是 React 16.8 引入的新特性...
```

### 2. 工具调用

AI 可以调用以下工具：

**Read** - 读取文件内容
```
[工具] 🔍 Read src/App.tsx
import React from 'react';
function App() {
  return <div>Hello</div>;
}
```

**Write** - 写入文件
```
[工具] ✏️ Write src/utils.ts
export function formatDate(date: Date) {
  return date.toLocaleString();
}
```

**Edit** - 编辑文件
```
[工具] ✏️ Edit src/App.tsx
- return <div>Hello</div>;
+ return <div>Hello, World!</div>;
```

### 3. 代码编辑器

**特性**：
- Monaco Editor（VS Code 同款）
- 语法高亮
- 代码补全
- 错误提示
- 多光标编辑

**使用方法**：
1. 点击文件树中的文件
2. 在右侧编辑器中编辑
3. 自动保存（或 `Ctrl+S` 手动保存）

### 4. 文件树

**功能**：
- 浏览项目结构
- 展开折叠目录
- 点击打开文件
- 右键菜单（重命名、删除等）

**快捷操作**：
- 单击：打开文件
- 双击：在编辑器中聚焦
- 右键：显示上下文菜单

## 🔧 高级功能

### 1. 多会话管理

**创建新会话**：
- 点击左上角 "新建对话" 按钮
- 或使用快捷键 `⌘N` / `Ctrl+N`

**切换会话**：
- 点击左侧会话列表
- 会话状态自动保存

**删除会话**：
- 右键点击会话 → 删除
- 会话历史永久删除

### 2. 消息持久化

**自动保存**：
- 每100条消息自动保存到 localStorage
- 应用重启后自动恢复

**手动清理**：
- 设置 → 清除历史
- 或删除会话

### 3. 错误恢复

**自动重连**：
- CLI 崩溃后5秒内自动重启
- 消息自动重发（最多3次）
- 会话状态不丢失

**错误提示**：
- 网络错误：显示重试按钮
- 超时错误：自动指数退避重试
- 权限错误：提示检查 API Key

### 4. 大文件处理

**自动分块**：
- 大文件（>5MB）自动分块读取
- 流式传输，不占用内存
- 实时显示加载进度

**性能优化**：
- 会话历史限制100条
- 编辑器延迟加载大文件
- 内存占用 <500MB

## ❓ 常见问题

### Q1: 如何配置 ClaudeCode CLI？

**步骤**：
1. 打开设置（⌘, / Ctrl+,）
2. 找到 "CLI 路径" 配置
3. 输入 ClaudeCode CLI 的完整路径
   - macOS: `/usr/local/bin/claude`
   - Linux: `/usr/bin/claude`
   - Windows: `C:\Program Files\ClaudeCode\claude.exe`
4. 点击"保存"并重启应用

### Q2: API Key 如何配置？

**方式1：环境变量**
```bash
export ANTHROPIC_AUTH_TOKEN="sk-ant-xxx"
```

**方式2：配置文件**
编辑 `~/.claude/settings.json`：
```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "sk-ant-xxx"
  }
}
```

**方式3：应用内设置**
1. 打开设置
2. 找到 "API Key" 配置
3. 输入你的 API Key
4. 点击"保存"

### Q3: 连接失败怎么办？

**检查清单**：
1. ✅ API Key 是否正确配置
2. ✅ 网络连接是否正常
3. ✅ CLI 路径是否正确
4. ✅ 防火墙是否允许应用访问网络

**重试步骤**：
1. 点击"重试"按钮
2. 或重启应用（⌘Q / Ctrl+Q）
3. 检查日志：开发者工具 → Console

### Q4: 如何提升响应速度？

**优化建议**：
1. 使用更快的模型（如 Claude Sonnet）
2. 减少上下文长度
3. 关闭不必要的文件
4. 清理浏览器缓存

### Q5: 内存占用过高怎么办？

**解决方案**：
1. 清理会话历史（设置 → 清除历史）
2. 关闭不必要的会话
3. 重启应用
4. 调整会话历史限制（设置 → 最大消息数）

## ⚙️ 配置说明

### 应用配置

**位置**：`~/.claude/settings.json`

**完整配置示例**：
```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "sk-ant-api03-xxx",
    "ANTHROPIC_BASE_URL": "https://api.anthropic.com",
    "MODEL": "claude-sonnet-4-20250514"
  },
  "permissions": {
    "allow": ["Bash", "Read", "Write", "Edit"],
    "deny": []
  },
  "enabledPlugins": {
    "github": true,
    "gitlab": false
  },
  "outputStyle": "engineer-professional",
  "maxHistorySize": 100
}
```

### 环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `ANTHROPIC_AUTH_TOKEN` | API密钥 | - |
| `ANTHROPIC_BASE_URL` | API基础URL | https://api.anthropic.com |
| `MODEL` | 使用的模型 | claude-opus-4-20250514 |
| `MAX_TOKENS` | 最大token数 | 4096 |

### 输出风格

**可选风格**：
- `engineer-professional`：工程师专业版（默认）
- `concise`：简洁版
- `detailed`：详细版
- `tutorial`：教程版

**设置方法**：
```json
{
  "outputStyle": "engineer-professional"
}
```

## 🎓 进阶使用

### 自定义主题

**创建主题**：
```css
/* custom-theme.css */
:root {
  --bg-primary: #1a1a1a;
  --text-primary: #ffffff;
  --accent-color: #c96442;
}
```

**应用主题**：
1. 打开设置
2. 选择"主题" → "自定义"
3. 上传 CSS 文件

### 插件开发

**插件示例**：
```typescript
// my-plugin.ts
export default {
  name: 'my-plugin',
  version: '1.0.0',
  activate() {
    console.log('Plugin activated');
  },
  deactivate() {
    console.log('Plugin deactivated');
  }
};
```

**安装插件**：
1. 将插件文件放到 `plugins/` 目录
2. 重启应用
3. 在设置中启用插件

### 脚本自动化

**批量操作**：

```javascript
// scripts/batch-process.js
const sessions = ['session1', 'session2', 'session3'];

sessions.forEach(sessionId => {
  // 批量处理会话
  processSession(sessionId);
});
```

## 📈 性能监控

### 查看性能指标

**打开开发者工具**：
1. `⌘Option+I` / `Ctrl+Shift+I`
2. 切换到 "Performance" 标签
3. 点击"开始录制"
4. 执行操作
5. 停止录制并分析

### 性能优化建议

1. **减少会话数量**：关闭不需要的会话
2. **限制历史长度**：设置最大100条
3. **使用更快的模型**：Sonnet 比 Opus 快
4. **启用缓存**：设置中启用"响应缓存"
