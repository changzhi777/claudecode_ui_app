## 🛠️ 开发环境搭建

### 系统要求

- **Node.js**: >= 18.0.0
- **pnpm**: >= 8.0.0
- **Git**: >= 2.30.0
- **操作系统**: macOS / Linux / Windows

### 初始化项目

```bash
# 1. 克隆仓库
git clone https://github.com/your-org/claudecode-ui-app.git
cd claudecode-ui-app

# 2. 安装依赖
pnpm install

# 3. 启动开发模式
pnpm dev

# 4. 运行测试
pnpm test

# 5. 类型检查
pnpm type-check

# 6. 代码检查
pnpm lint
```

### VS Code 配置

**推荐扩展**：
- ESLint
- Prettier
- TypeScript Vue Plugin (Volar)
- ESLint + TypeScript

**settings.json**：
```json
{
  "typescript.tsdk": "node_modules/typescript/lib",
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  }
}
```

## 🔧 核心模块

### 1. 主进程模块

**职责**：
- 管理 CLI 进程池
- 处理 IPC 通信
- 文件系统操作
- 窗口管理

**关键文件**：
- `src/main/index.ts` - 主入口
- `src/main/cli/ProcessPool.ts` - 进程池
- `src/main/cli/CLIProcess.ts` - CLI 进程封装
- `src/main/ipc/*` - IPC 处理器

**示例代码**：
```typescript
// src/main/index.ts
import { app, BrowserWindow } from 'electron';
import { IPCServer } from '../ipc-bridge/ipcServer';

app.whenReady().then(() => {
  const window = createWindow();
  new IPCServer();
});
```

### 2. 渲染进程模块

**职责**：
- UI 渲染
- 用户交互
- 状态管理
- Worker 通信

**关键文件**：
- `src/renderer/main.tsx` - React 入口
- `src/renderer/modules/*` - 功能模块
- `src/renderer/stores/*` - 状态管理
- `src/renderer/workers/*` - Worker 线程

**示例代码**：
```typescript
// src/renderer/main.tsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import { App } from './App';

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
```

### 3. IPC 通信模块

**职责**：
- 主进程与渲染进程通信
- 类型安全的 IPC 调用
- 错误处理

**关键文件**：
- `src/ipc-bridge/channels.ts` - 频道定义
- `src/ipc-bridge/main-handlers.ts` - 主进程处理器
- `src/ipc-bridge/renderer-invokers.ts` - 渲染进程调用器

**示例代码**：
```typescript
// 定义频道
export const IPC_CHANNELS = {
  CLI_START: 'cli:start',
  CLI_STOP: 'cli:stop',
  FILE_READ: 'file:read',
} as const;

// 主进程处理器
ipcMain.handle(IPC_CHANNELS.FILE_READ, async (event, filePath) => {
  const content = await fs.promises.readFile(filePath, 'utf-8');
  return { success: true, data: content };
});
```

### 4. 状态管理模块

**职责**：
- 全局状态管理
- 会话持久化
- 状态订阅

**关键文件**：
- `src/renderer/stores/cliStore.ts` - CLI 状态
- `src/renderer/stores/editorStore.ts` - 编辑器状态
- `src/renderer/stores/uiStore.ts` - UI 状态

**示例代码**：
```typescript
// src/renderer/stores/cliStore.ts
import { create } from 'zustand';

export const useCLIStore = create<CLIStoreState>((set) => ({
  sessionId: null,
  isConnected: false,
  messages: [],

  initialize: async (sessionId) => {
    // 初始化逻辑
  },

  sendMessage: async (content) => {
    // 发送消息逻辑
  },
}));
```

### 5. Worker 线程模块

**职责**：
- 处理 CLI 通信
- JSON 解析
- 错误恢复

**关键文件**：
- `src/renderer/workers/cli-worker.ts` - CLI Worker
- `src/renderer/services/WorkerManager.ts` - Worker 管理器

**示例代码**：
```typescript
// src/renderer/workers/cli-worker.ts
self.onmessage = async (e: MessageEvent<WorkerMessage>) => {
  const { type, payload } = e.data;

  switch (type) {
    case 'INITIALIZE':
      await initialize(payload);
      postMessage({ type: 'INITIALIZED' });
      break;
    case 'SEND_MESSAGE':
      await sendMessage(payload);
      break;
  }
};
```

## 🔄 开发工作流

### Git 工作流

```bash
# 1. 创建功能分支
git checkout -b feature/your-feature

# 2. 开发并提交
git add .
git commit -m "feat: add your feature"

# 3. 推送到远程
git push origin feature/your-feature

# 4. 创建 Pull Request
# 在 GitHub 上创建 PR
```

### Commit 规范

**格式**：
```
<type>(<scope>): <subject>
```

**类型**：
- `feat`: 新功能
- `fix`: 修复 Bug
- `docs`: 文档更新
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

**示例**：
```bash
git commit -m "feat(cli): add process pool management"
git commit -m "fix(worker): handle connection timeout"
git commit -m "docs: update developer guide"
```

### 代码审查清单

**提交前检查**：
- [ ] 代码通过 `pnpm type-check`
- [ ] 代码通过 `pnpm lint`
- [ ] 代码通过 `pnpm test`
- [ ] 添加了必要的测试
- [ ] 更新了相关文档
- [ ] Commit 信息符合规范

```typescript

  it('应该初始化 Worker', async () => {
```

### 代码规范

**TypeScript 规范**：
```typescript
// ✅ 好的实践
interface UserData {
  id: string;
  name: string;
  email: string;
}

async function getUser(id: string): Promise<UserData> {
  const response = await fetch(`/api/users/${id}`);
  return response.json();
}

// ❌ 避免的做法
async function getUser(id) { // 缺少类型
  return await fetch('/api/users/' + id); // 使用模板字符串
}
```

**React 规范**：
```typescript
// ✅ 好的实践
interface ButtonProps {
  label: string;
  onClick: () => void;
  disabled?: boolean;
}

export const Button: React.FC<ButtonProps> = ({
  label,
  onClick,
  disabled = false,
}) => {
  return (
    <button onClick={onClick} disabled={disabled}>
      {label}
    </button>
  );
};

// ❌ 避免的做法
export const Button = (props: any) => { // 使用 any 类型
  return <button onclick={props.onClick}>{props.label}</button>; // 小写 onclick
};
```
